ALGORITMA PENGURUTAN LANJUTAN

1. Merge Sort

Konsep Dasar: Permasalahan pengurutan akan jauh lebih mudah dilaksanakan jika hanya terdapat satu elemen saja di dalam list data, karena data tunggal otomatis sudah berada dalam keadaan terurut. Merge Sort memanfaatkan konsep rekursif ini dan dibagi menjadi dua tahapan utama:

  1. Splitting / Divide (Top-Down Process): Proses membagi list data menjadi dua buah bagian (kiri dan kanan) secara terus-menerus hingga hanya tersisa satu elemen tunggal di setiap sub-list.
  2. Merging (Bottom-Up Process): Proses menggabungkan kembali sub-list yang telah terpecah. Proses ini melibatkan pembandingan elemen; jika suatu data lebih kecil, ia diletakkan pada indeks yang lebih rendah sehingga hasil penggabungannya otomatis terurut.

2. Quick Sort

Konsep Dasar: Memiliki kesamaan prinsip kerja dengan Merge Sort, yaitu menggunakan proses divide (pembagian) dan combine (penggabungan). Namun, terdapat perbedaan mendasar pada syarat pembagiannya.

Tahapan Kerja:

  1. Divide (Partisi berdasarkan Pivot): Pada Quick Sort, pembagian data menjadi dua bagian harus memenuhi ketentuan bahwa sub-list sebelah kiri berisi data yang nilainya lebih kecil atau sama dengan nilai acuan (pivot), sedangkan sub-list sebelah kanan berisi data yang nilainya lebih besar dari nilai pivot. Nilai pivot yang dipilih biasanya merupakan data pada indeks awal dari barisan yang akan dipartisi.
  2. Combine: Berbeda dengan Merge Sort yang melakukan pengurutan saat menggabungkan data (merging), Quick Sort melakukan pengurutan secara langsung selama proses partisi (divide). Di akhir proses partisi, elemen pivot akan menempati posisi indeks yang sebenarnya dalam keadaan terurut. Langkah selanjutnya tinggal memerintahkan fungsi partisi secara berulang pada sub-list sebelum pivot dan sub-list setelah pivot hingga seluruh elemen terurut.

3. Counting Sort

  • Konsep Dasar: Counting Sort merupakan algoritma pengurutan yang memiliki waktu komputasi lebih rendah dibandingkan dengan algoritma pembandingan tingkat lanjut seperti Merge Sort dan Quick Sort.
  • Cara Kerja: Algoritma ini tidak membandingkan elemen satu per satu, melainkan menghitung total frekuensi kemunculan (jumlah elemen) dari masing-masing data terlebih dahulu. Berdasarkan jumlah elemen yang telah diketahui tersebut, nilai akumulatif (kumulatif) akan dihitung untuk menentukan posisi indeks akhir yang tepat dan absolut bagi setiap elemen data di dalam list hasil.
  • Contoh Kasus: Misalkan terdapat data [4, 3, 1, 2, 4, 2, 1, 1], perhitungannya adalah:
    • Data 1 berjumlah 3 buah, maka akan menempati indeks ke-0 sampai ke-2.
    • Data 2 berjumlah 2 buah, maka akan menempati indeks ke-3 sampai ke-4.
    • Data 3 berjumlah 1 buah, maka akan menempati indeks ke-5.
    • Data 4 berjumlah 2 buah, maka akan menempati indeks ke-6 sampai ke-7.

3. Implementasi Counting Sort

Kode ini bekerja menggunakan tiga tahapan loop: menyiapkan array penghitung C, menghitung frekuensi serta nilai kumulatif, dan menyusun hasil akhir ke dalam array B.

def countingSort(A, k): C = [] B = [] # Loop 1: Menyiapkan list C untuk menyimpan jumlah frekuensi dengan inisialisasi 0 for i in range(k + 1): C.append(0) # Loop 2: Menghitung jumlah elemen/frekuensi masing-masing data dari array A for j in range(len(A)): C[A[j]] = C[A[j]] + 1 B.append(0) # Sekaligus menyiapkan kerangka dimensi list hasil B # Menghitung nilai akumulasi komulatif dari list frekuensi C for i in range(1, k + 1): C[i] = C[i] + C[i - 1] # Loop 3: Menempatkan data ke dalam indeks yang tepat pada list B secara mundur for j in range(len(A) - 1, -1, -1): B[C[A[j]] - 1] = A[j] C[A[j]] = C[A[j]] - 1 return B # Contoh Eksekusi (93 melambangkan nilai data tertinggi / k di dalam list) a_list = [54, 26, 93, 17, 1, 31, 2, 55, 20, 93, 20, 3, 4, 5] hasil_urut = countingSort(a_list, 93) print("Hasil Counting Sort:", hasil_urut)
