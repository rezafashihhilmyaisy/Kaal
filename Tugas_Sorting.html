Analisis Perbandingan Waktu Eksekusi Algoritma Sorting

Analisis mengenai perbandingan waktu eksekusi algoritma sorting sangat menarik, terutama untuk melihat bagaimana perbedaan kompleksitas waktu (Time Complexity) memengaruhi performa di dunia nyata.

Meskipun data berukuran 100 hingga 500 cenderung kecil untuk komputer modern saat ini—sehingga perbedaannya mungkin dalam hitungan milidetik (milliseconds) atau bahkan mikrodetik (microseconds)—tren pertumbuhannya akan terlihat sangat jelas.

1. Kelompok Algoritma Lambat (Bubble, Selection, Insertion)

Pada jumlah data yang relatif kecil (100-500), kelompok ini masih bisa mengeksekusi data dengan cepat, namun grafik pertambahan waktunya akan melengkung tajam ke atas (eksponensial / kuadratik).

  • Bubble Sort: Hampir selalu menjadi yang paling lambat. Algoritma ini melakukan banyak sekali operasi pertukaran (swap) data di setiap langkahnya.
  • Selection Sort: Biasanya sedikit lebih cepat daripada Bubble Sort karena jumlah pertukaran (swap) datanya diminimalisir (maksimal hanya n kali), meskipun jumlah perbandingannya tetap tinggi.
  • Insertion Sort: Merupakan yang terbaik di kelompok ini. Jika data acaknya kebetulan agak terurut, Insertion Sort bekerja sangat efisien. Pada data 100, ia bahkan bisa bersaing dengan kelompok cepat.

2. Kelompok Algoritma Cepat (Merge, Quick)

Kelompok ini menggunakan pendekatan Divide and Conquer (bagi dan kuasai). Pertumbuhan waktu eksekusinya cenderung landai (linear-logaritmik).

  • Quick Sort: Pada data acak, Quick Sort biasanya menjadi yang paling cepat di antara semuanya karena faktor konstanta operasinya di memori yang sangat kecil.
  • Merge Sort: Sangat stabil dan konsisten di setiap kondisi data. Sedikit lebih lambat dari Quick Sort pada data acak karena Merge Sort membutuhkan memori tambahan untuk melakukan proses penggabungan (merge).
import random import time # ========================================================= # BUBBLE SORT # ========================================================= def bubble_sort(arr): data = arr.copy() for i in range(len(data)): for j in range(0, len(data)-i-1): if data[j] > data[j+1]: data[j], data[j+1] = data[j+1], data[j] return data # ========================================================= # SELECTION SORT # ========================================================= def selection_sort(arr): data = arr.copy() for i in range(len(data)): min_index = i for j in range(i+1, len(data)): if data[j] < data[min_index]: min_index = j data[i], data[min_index] = data[min_index], data[i] return data # ========================================================= # INSERTION SORT # ========================================================= def insertion_sort(arr): data = arr.copy() for i in range(1, len(data)): key = data[i] j = i - 1 while j >= 0 and key < data[j]: data[j+1] = data[j] j -= 1 data[j+1] = key return data # ========================================================= # MERGE SORT # ========================================================= def merge_sort(arr): if len(arr) <= 1: return arr mid = len(arr) // 2 left = merge_sort(arr[:mid]) right = merge_sort(arr[mid:]) return merge(left, right) def merge(left, right): result = [] i = 0 j = 0 while i < len(left) and j < len(right): if left[i] < right[j]: result.append(left[i]) i += 1 else: result.append(right[j]) j += 1 result.extend(left[i:]) result.extend(right[j:]) return result # ========================================================= # QUICK SORT # ========================================================= def quick_sort(arr): if len(arr) <= 1: return arr pivot = arr[len(arr)//2] left = [x for x in arr if x < pivot] middle = [x for x in arr if x == pivot] right = [x for x in arr if x > pivot] return quick_sort(left) + middle + quick_sort(right) # PENGUJIAN WAKTU SORTING ukuran_data = [100, 300, 500] for ukuran in ukuran_data: print("============================================================") print("Jumlah Data :", ukuran) data = [random.randint(1, 1000) for _ in range(ukuran)] # Bubble Sort start = time.time() bubble_sort(data) end = time.time() print("Bubble Sort :", end - start, "detik") # Selection Sort start = time.time() selection_sort(data) end = time.time() print("Selection Sort :", end - start, "detik") # Insertion Sort start = time.time() insertion_sort(data) end = time.time() print("Insertion Sort :", end - start, "detik") # Merge Sort start = time.time() merge_sort(data) end = time.time() print("Merge Sort :", end - start, "detik") # Quick Sort start = time.time() quick_sort(data) end = time.time() print("Quick Sort :", end - start, "detik")

Kesimpulan Analisis

Jika Anda menjalankan program tersebut dan memplot hasilnya ke dalam sebuah grafik, Anda akan melihat fenomena berikut:

  1. Pada 100 Data: Selisih waktu antar algoritma akan terasa sangat tipis, bahkan mungkin mendekati 0 milidetik pada arsitektur komputer modern.
  2. Pada 300 & 500 Data: Jarak performa mulai melebar secara signifikan. Waktu eksekusi Bubble Sort dan Selection Sort melonjak tajam. Hal ini sesuai dengan karakteristik kompleksitas kuadratik $O(n^2)$, di mana jika jumlah data naik 5x lipat (dari 100 ke 500), waktu komputasi kasarnya akan meningkat sekitar 25x lipat. Sebaliknya, grafik Quick Sort dan Merge Sort ($O(n \log n)$) akan tetap terlihat landai dan hampir mendatar.
